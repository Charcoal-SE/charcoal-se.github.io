API Documentation (V2)

This page contains documentation for version 2.0 of the metasmoke API. If that’s not what you’re expecting, you’re in the wrong place.

Authentication

Like 1.0, you need a key to access the API. Ping an admin to get one if you don’t already have one. If you’ve previously been given a key but no longer have it, you should be able to access it at https://metasmoke.erwaysoftware.com/api/keys/mine.

For write authentication, the same flow applies as for v1:

  • Send your user to https://metasmoke.erwaysoftware.com/oauth/request?key=YOUR_KEY_HERE.
  • If the user authorizes your app, they will be given a code to provide you. Ask them to enter this code, then:
  • Send a HTTP GET request to https://metasmoke.erwaysoftware.com/oauth/token?key=YOUR_KEY_HERE&code=THEIR_CODE_HERE. The response is a JSON object containing one key, token, the value of which is your write token. Include this as the token parameter on any requests that require it.

Filters

Filters have been totally reworked in V2 of the API; they really were that terrible in v1. Again, visit https://metasmoke.erwaysoftware.com/api/filters to create a filter, and include it as the filter parameter on any requests you need it for.

Filters are not quite immutable. If the server configuration remains the same, then the same filter will always represent the same fields. However, if the server configuration changes - such as to add a new field to the API - then filters will also change. If you’re getting an SQL error on your requests, this is probably the cause - go recalculate your filters.

You can calculate filters programmatically if you wish: send a POST request to https://metasmoke.erwaysoftware.com/api/filters with the fields parameter set to an array of field names that you want to include (i.e. ['announcements.id', 'announcements.text', 'announcements.expiry']), and you will get back a JSON object containing the filter.

You may also include the filter-generation logic directly in your application if you really want; it’s less complex than v1’s, so that’s not too much of a liability. Bear in mind, however, that if the metasmoke implementation changes you will need to push out an update to your implementation to match.

Routes

Routes marked with [W] are write routes; you must use a POST request and present a write token for these routes.

Path Description
Announcements  
/api/v2.0/announcements List all announcements.
/api/v2.0/announcements/active List active (non-expired) announcements.
/api/v2.0/announcements/expired List expired announcements.
/api/v2.0/announcements/create [W] Create an announcement. Requires params: text and expiry. Requires core role.
Apps  
/api/v2.0/apps List all apps (API keys, technically).
Commit Statuses  
/api/v2.0/commits List all commit statuses.
Debug  
/api/v2.0/debug Return debugging information for the API. Not intended for widespread use, more for use while developing this API.
Deletion Logs  
/api/v2.0/deletions List all deletion logs.
/api/v2.0/deletions/post/:id List deletion logs on the post identified by id.
Domain Tags  
/api/v2.0/tags List all domain tags.
/api/v2.0/tags/name/:name List tags with a name of name.
/api/v2.0/tags/:id/domains List domains associated with the tag identified by id.
Feedbacks  
/api/v2.0/feedbacks List all feedbacks.
/api/v2.0/feedbacks/post/:id List feedbacks on the post identified by id.
/api/v2.0/feedbacks/user/:id List feedbacks created by user id.
/api/v2.0/feedbacks/app/:id List feedbacks created from the app identified by id.
/api/v2.0/feedbacks/post/:id/create [W] Create a feedback on the post identified by id. Requires params: type. Requires reviewer role.
Posts  
/api/v2.0/posts List all posts.
/api/v2.0/posts/between List posts created between a specified start and end date. Requires params: from and to, in ISO 8601 format (2017-12-25T00:00:00.000000Z).
/api/v2.0/posts/site List posts on a particular site. Requires params: site (e.g. site=stackoverflow.com).
/api/v2.0/posts/urls List posts identified by particular URLs. Requires params: urls - a comma-separated list of URLs.
/api/v2.0/posts/feedback List posts that have a certain type of feedback on them. Requires params: type.
/api/v2.0/posts/undeleted List posts that have not been recorded as deleted.
/api/v2.0/posts/report [W] Report a post to Smokey. Requires params: url. Requires reviewer role.
/api/v2.0/posts/:ids List posts identified by their IDs. ids is a comma-separated list of IDs.
/api/v2.0/posts/:id/reasons List the reasons associated with the post identified by id.
/api/v2.0/posts/:id/domains List the domains associated with the post identified by id.
/api/v2.0/posts/:id/flag [W] Cast a spam flag on the post identified by id. Requires reviewer role.
Reasons  
/api/v2.0/reasons List all reasons.
/api/v2.0/reasons/:ids List reasons identified by their IDs. ids is a comma-separated list of IDs.
/api/v2.0/reasons/:id/posts List the posts that have been caught by the reason identified by id.
Smoke Detectors  
/api/v2.0/smokeys List all Smoke Detectors.
Spam Domains  
/api/v2.0/domains List all spam domains.
/api/v2.0/domains/:id/posts List posts that contain the domain identified by id.
/api/v2.0/domains/:id/tags/add [W] Add a tag to a domain. Requires params: tag. Requires core role.
